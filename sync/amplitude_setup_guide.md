## 1단계: Amplitude 계정 생성 및 프로젝트 만들기

1.  **Amplitude 접속:** 웹 브라우저에서 [amplitude.com](https://amplitude.com/)에 접속합니다.
2.  **계정 생성:**
    *   우측 상단의 'Get Started' 또는 'Sign Up' 버튼을 클릭합니다.
    *   필요한 정보를 입력하여 새 계정을 만듭니다. (회사 이메일 사용 권장)
3.  **로그인:** 계정 생성 후 로그인합니다.
4.  **조직(Organization) 생성:**
    *   최초 로그인 시, 조직(Organization)을 만들라는 안내가 나옵니다.
    *   회사 또는 팀 이름을 입력하여 조직을 생성합니다.
5.  **프로젝트(Project) 생성:**
    *   조직 생성 후, 첫 프로젝트를 만들게 됩니다.
    *   프로젝트 이름(예: `PHM Service`)을 입력하고 'Create Project' 버튼을 클릭합니다.
    *   이제 데이터를 분석할 공간이 마련되었습니다.

## 2단계: API 키 발급 및 확인하기

API 키는 F/E, B/E 개발자가 트래킹 코드를 심을 때 사용하는 고유 식별자입니다.

1.  **설정 메뉴로 이동:**
    *   화면 좌측 하단의 **'Settings'** 톱니바퀴 아이콘을 클릭합니다.
    *   **'Projects'** 메뉴를 선택하고, 방금 생성한 프로젝트(예: `PHM Service`)를 클릭합니다.
2.  **API 키 확인:**
    *   프로젝트 설정 화면 중앙에 **'API Key'** 와 **'Secret Key'** 가 표시됩니다.
    *   **`API Key`** 를 복사하여 F/E 개발자에게 전달합니다. (Next.js에서는 `NEXT_PUBLIC_AMPLITUDE_API_KEY` 로 사용됩니다)
    *   **`Secret Key`** 는 B/E 개발자에게 전달합니다. (`payment_complete` 이벤트 전송 시 필요)

    > **보안 주의:** 이 키들은 외부에 유출되지 않도록 안전하게 전달해야 합니다.

## 3단계: 이벤트 분류(Taxonomy) 계획하기

아직 데이터가 들어오지 않았지만, 어떤 이벤트를 추적할지 Amplitude에 미리 알려주는 과정입니다. `sync.ko.md` 파일의 내용을 기반으로 합니다.

1.  **Taxonomy 메뉴 이동:**
    *   좌측 메뉴 바에서 **'Data'** 섹션을 클릭합니다.
    *   드롭다운 메뉴에서 **'Taxonomy'** 를 선택합니다.
2.  **이벤트(Events) 추가:**
    *   'Events' 탭을 선택하고, 우측 상단의 **'Add Event'** 버튼을 클릭합니다.
    *   `sync.ko.md` 파일의 **1단계**에 나열된 13개 이벤트를 하나씩 추가합니다.
        *   `landing_enter`
        *   `intro_step_complete`
        *   ...
        *   `unselected_character_click`
    *   **Display Name:** 이벤트 이름을 그대로 입력합니다.
    *   **Category:** 지금은 비워두거나, 'Core' 같은 카테고리를 만들어 분류할 수 있습니다.
    *   **Description:** 각 이벤트의 목적을 간략히 설명합니다. (예: `landing_enter` - "서비스 진입 및 유입 채널 분석")

3.  **이벤트 속성(Properties) 추가 (선택 사항이지만 권장):**
    *   이벤트를 추가할 때, 해당 이벤트가 가질 수 있는 속성(Properties)도 함께 정의할 수 있습니다.
    *   예를 들어, `character_select` 이벤트를 추가할 때:
        *   'Event Properties' 섹션에서 'Add Property' 클릭
        *   **Property Name:** `character_id`
        *   **Type:** `String`
        *   **Description:** "선택한 캐릭터의 ID (A 또는 B)"
    *   `sync.ko.md` 파일의 **2단계**에 있는 코드들을 참고하여 각 이벤트에 필요한 속성들을 미리 정의해두면 데이터가 들어왔을 때 분석이 훨씬 쉬워집니다.

## 4단계: Funnel Chart (퍼널 차트) 생성하기

사용자의 특정 여정(Funnel)을 미리 만들어 둡니다. 데이터가 없어도 '틀'을 만들 수 있습니다.

1.  **차트 생성 메뉴:**
    *   좌측 상단의 **'+ New'** 버튼을 클릭하고, **'Analysis'** > **'Funnel'** 을 선택합니다.
2.  **[HMDA-1] 활성화 측정 퍼널 만들기:**
    *   **Events:** 상단의 'Select event...' 드롭다운을 클릭하여 이벤트를 순서대로 추가합니다.
        1.  `landing_enter`
        2.  `enter_character_select`
        3.  `character_select`
    *   **차트 이름 저장:** 우측 상단의 'Untitled Funnel'을 클릭하여 "활성화 퍼널 (Activation Funnel)" 과 같이 알아보기 쉬운 이름으로 저장합니다.
3.  **[HMDA-3] 유료 전환 측정 퍼널 만들기:**
    *   위와 동일한 방법으로 새 퍼널 차트를 만듭니다.
    *   **Events:**
        1.  `result_received`
        2.  `payment_click`
        3.  `payment_complete`
    *   **차트 이름 저장:** "유료 전환 퍼널 (Payment Conversion Funnel)" 로 저장합니다.

## 5단계: Segmentation Chart (세그먼테이션 차트) 생성하기

특정 이벤트의 발생 횟수나 속성별 분포를 볼 수 있는 차트입니다.

1.  **차트 생성 메뉴:**
    *   좌측 상단의 **'+ New'** 버튼을 클릭하고, **'Analysis'** > **'Segmentation'** 을 선택합니다.
2.  **[HMDA-3] 캐릭터별 결제 완료 수 비교 차트 만들기:**
    *   **Events:** 상단의 'Select event...'에서 `payment_complete` 이벤트를 선택합니다.
    *   **Group by:** 우측의 '... grouped by'에서 `character_id` 속성을 선택합니다. (Taxonomy에 미리 정의해두었다면 쉽게 찾을 수 있습니다.)
    *   **차트 시각화:** 막대 차트(Bar Chart)나 파이 차트(Pie Chart)로 변경하여 시각적으로 비교하기 쉽게 만듭니다.
    *   **차트 이름 저장:** "캐릭터별 결제 완료 수" 와 같이 저장합니다.

## 6단계: 대시보드 구성하기

위에서 만든 여러 차트들을 한 곳에 모아놓는 공간입니다.

1.  **대시보드 생성:**
    *   좌측 상단의 **'+ New'** 버튼을 클릭하고, **'Dashboard'** 를 선택합니다.
    *   대시보드 이름을 "PHM 핵심 지표" 등으로 지정하고 생성합니다.
2.  **차트 추가:**
    *   대시보드 우측 상단의 **'Add to Dashboard'** 버튼을 클릭합니다.
    *   **'Charts'** 탭에서 이전에 저장해둔 차트("활성화 퍼널", "유료 전환 퍼널", "캐릭터별 결제 완료 수" 등)들을 선택하여 대시보드에 추가합니다.
    *   차트 크기와 위치를 자유롭게 조정하여 보기 좋게 배치합니다.

---

## 다음 단계

이제 Amplitude 대시보드의 '틀'이 모두 준비되었습니다.

*   개발자에게 **2단계**에서 발급받은 **API 키**를 전달하여 이벤트 트래킹 코드 개발을 요청합니다.
*   개발이 완료되고 실제 사용자 데이터가 Amplitude로 전송되기 시작하면, 이 대시보드에 자동으로 데이터가 채워지는 것을 확인할 수 있습니다.
*   데이터가 들어온 후에는 각 차트의 기간을 설정하거나, 특정 사용자 그룹(Segment)을 필터링하여 더 깊이 있는 분석을 수행할 수 있습니다.
