## 1. 용어표

| 용어 / 기호 | 의미 | 설명 / 범위 |
| --- | --- | --- |
| $C$ | Community Signal | 트위터·아카라이브 등 서브컬처 커뮤니티 내 사주·운세 관련 대화 및 시딩 콘텐츠 노출. 행동 흐름의 시작점 |
| $C_{community}$ | Community Discourse | 커뮤니티 내 자발적으로 발생하는 사주·운세 관련 대화 신호. $C$의 구체적 관측 형태 |
| $I$ | Subculture Interest | 커뮤니티 노출 누적으로 형성된 사주·운세 탐색 관심도. $C$ 노출 반복으로 활성화 |
| $I_{adopt}$ | Saju Adopt Index | 사주·운세에 대한 탐색·채택 욕구. 서브컬처 사용자도 독립적으로 보유하고 있으나 적합한 공급처를 찾지 못한 상태 |
| $I_{subculture}$ | Subculture Identity Index | 서브컬처 감성 정체성 지수. 덕후·팬덤 문화에 깊이 속해 있으며 콘텐츠 소비·생산 방식이 메인스트림과 구별되는 사용자군의 특성. $M$ 형성 가능성과 비례 |
| $O$ | Observer State | 콘텐츠를 소비하지만 행동($A$)은 하지 않고 판단만 지속하는 상태. Bootstrap 전환 실패의 80%가 여기서 발생. Bootstrap 핵심 목표: $O→A_1$ |
| $M$ | Meaning | 플랫폼이 제공하는 의미 해석 레이어. "이건 내 감성 공간이다"라는 인식. $A_1$ 전이의 직접 조건. $J$ 이전에 형성됨. $M→J$ 연결 실패 = Semantic Mapping 실패 |
| $A$ | Action | 실제 행동. $A_1$, $A_2$, $A_3$으로 분해. 단일 상태로 묶지 않음 |
| $A_1$ | First Action | 결과 보기 — 저비용 첫 행동. curiosity로 발생. Test 2 핵심 |
| $A_2$ | Revisit Action | 재방문 — 정체성 기반 행동. "이 서비스, 또 왔다"는 것 자체가 $J_{initial}$ 신호. identity resonance로 발생. MVP 단계 Test 4 핵심 |
| $A_3$ | Pay Action | 결제 — 가치 인정 행동. utility + trust로 발생. Test 3 핵심 |
| $A_{share}$ | Share Action | 결제 완료 사용자의 공유 카드 공유 행동 |
| $J$ | Belief State | "이 플랫폼이 나와 내 서브컬처 정체성을 위한 공간이다"라는 내부 신념 상태. 반복 행동 후 점진적으로 형성되는 누적 상태. $J_{t+1}=F(J_t,A_t,M_t)$로 업데이트. Bootstrap 이후 형성 |
| $B_{brand}$ | Brand Mismatch Barrier | 기존 플랫폼(청월당 등)이 자신의 서브컬처 정체성과 맞지 않는다는 심리적 마찰. 기능 문제가 아닌 감성·공간의 문제 |
| $B_{supply}$ | Supply Barrier | 서브컬처 맥락에 맞는 콘텐츠 공급 부재 |
| $B_{stigma}$ | Stigma Barrier | 사주 본다는 사실 노출 부담 + AI 창작물 거부감 등 취향 노출에 대한 심리적 저항 |
| $B_{total}$ | Total Entry Barrier | 진입 장벽 총합. $B_{total}=B_{brand}+B_{supply}+B_{stigma}$. $P(A)∼e^{-B_{total}}$ |
| $U$ | Community Uncertainty | 커뮤니티 내 감정적 긴장 상태. 아이돌 컴백·커뮤니티 이슈·시즌 이벤트 등 $I_{adopt}$ 상승 외부 촉진 요인 |
| $TTR_1$ | Time-to-Result (First) | 진입 → $A_1$ 완료까지 소요 시간. ≤ 15초 목표 |
| $TTR_2$ | Time-to-Result (Meaning) | 결과 화면에서 $M$ 형성을 위한 의미 해석 체류 시간. ≥ 60초 목표 |
| $CTR$ | Click Through Rate | 시딩 콘텐츠 노출 대비 랜딩 진입 비율. Test 1 핵심 지표 |
| $CR_t$ | Completion Rate | 랜딩 진입 후 결과 수령($A_1$) 완료 비율. ≥ 50% 목표. Test 2 핵심 |
| $CTR_{pay}$ | Pay Button CTR | 페이월 노출 대비 결제 버튼 클릭 비율. ≥ 10% 목표. Test 3 |
| $CVR_{pay}$ | Pay Conversion Rate | 결제 버튼 클릭 후 완료 비율. ≥ 5% 목표. Test 3 핵심 |
| $SR_t$ | Share Rate | 결과 수령 사용자 중 공유 행동 발생 비율. ≥ 30% 목표. Test 4 핵심 |
| $UTM_{source}$ | 유입 채널 추적 파라미터 | 커뮤니티별 시딩 효과 분리 측정. 수집률 ≥ 0.95 |
| $D1/D7/D30$ | Retention | 재방문율. $J$ 형성 및 안정화 여부 간접 지표. Test 4 관찰 대상 |
| $ARPU$ | Average Revenue Per User | 사용자당 평균 수익. $₩990 × CVR_{pay}$ |
| $Fake J$ 리스크 | Fake Belief Risk | 행동($A$) 발생 → $J$ 형성으로 착각하는 오류. $A \neq J$ |

**상태 전이 구조 (DAG)**

```
C → I → O → M → A₁ → A₂ → A₃ → Aₛₕₐᵣₑ → J
```

Bootstrap 단계 핵심 전환: $O → A_1$
초기 행동 트리거 ($J$ 없는 상태): $A_1 = f(M, curiosity, friction, U)$

## 2. 문제 정의

### 사용자 관측 / Pain Point

서브컬처 감성 여성 사용자는 $I_{adopt}$를 보유하고 있으나, 기존 플랫폼의 $B_{brand}$ 자동 작동으로 진입 의도가 꺾이고, 진입하더라도 $M$ 형성 없이 $O$ 상태에 고착되어 $A_1$으로 전환되지 못하고 있다.

현재 이 Pain Point는 행동 데이터 없이 커뮤니티 모니터링 기반으로 추정된 상태이며, Bootstrap Test를 통해 실재 여부를 검증한다.

### 문제 배경

국내 사주 플랫폼이 메인스트림화되면서 서브컬처 감성 여성 사용자를 위한 공간이 구조적으로 비어가고 있다. 청월당의 더현대 입점 이후 해당 플랫폼은 $B_{brand}$를 자동 상승시키는 방향으로 이동했고, 트위터·아카라이브 등 커뮤니티 내 사주·운세 관련 대화 $C_{community}$는 이미 활성화되어 있으나 이를 포착해 $A_1$으로 연결하는 서비스 구조가 없다.

캐릭터 간의 관계성과 서사 몰입형 콘텐츠를 적극적으로 소비하는 여성 타겟은 현실의 불안을 해소할 사주 탐색 욕구($I_{adopt}$)를 강하게 보유하고 있으나, 기존의 단방향 정보 제공형 사주 서비스는 이들의 스토리텔링 및 감정적 교감 니즈를 전혀 충족시키지 못해 시장의 공백이 존재한다.

### 문제 정의

서브컬처 감성 여성 사용자가 $C_{community}$ 내에서 $I_{adopt}$를 보유하고 있음에도, $B_{total}(=B_{brand}+B_{supply}+B_{stigma})$이 임계값을 초과하여 Observer 상태($O$)에서 첫 행동($A_1$)으로 전환되지 못하고 있다.

| Test | 검증 대상 |
| --- | --- |
| Test 1 | $B_{brand}$ 존재 및 $C→I$ 가능성 |
| Test 2 | $O→A_1$ 전환 가능성 ⇒ $O→M→A_1$ |
| Test 3 | $A_3$(WTP) 발생 가능성 ⇒ $A_1→A_3$ |
| Test 4 | $A_2$(재방문) + $A_{share}$(공유) 발생 가능성 |

### 목표 (Objective)

서브컬처 감성 여성 사용자가 커뮤니티 자극($C$)에서 첫 결과 소비($A_1$)와 공유($A_2$)까지 도달하는 행동 흐름이 **실제로 발생하는가**를 확인한다.

- **단기**: $O→A_1$ 전환 존재 확인
- **중기**: $A_1→A_3$ WTP 존재 확인, $A_1→A_2$ 재방문 존재 확인
- **후기**: $CVR_{pay}$ 확인, $A_3→A_{share}$ 래퍼럴 루프 시작 확인

$J$(신념 상태) 형성은 Bootstrap 검증 대상이 아님. $A_2$(재방문)가 $J$의 초기 형태($A_2 \approx J_{initial}$)로 기능하는지를 간접 관찰하는 것으로 충분.

## 3. 전략 프레임워크 (OKR)

### Experiment Meta OKR

| Objective | Key Result | Result |
| --- | --- | --- |
| 서브컬처 감성 여성 사용자의 행동 흐름($C→I→O→M→A_1→A_2→A_3→A_{share}$)을 상태-이벤트 기반 전이 시스템으로 정의하고, 사용자 단위에서 재구성 가능한 시퀀스 데이터 구조로 정규화한다 | 상태 집합 $\{C, I, O, M, A_1, A_2, A_3, A_{share}\}$ 및 이벤트 집합 \{impression, interest, observe, action, revisit, purchase, share\}을 정의하고, 각 이벤트를 $(state_{from}, state_{to})$ 전이로 매핑한 event log schema (user_id, timestamp, event_type, $state_{from}$, $state_{to}$)를 구축한다 | 사용자 행동이 time-ordered raw event stream으로 누적되며, 각 사용자에 대해 상태 전이 시퀀스 재구성이 가능하고, 전이 그래프 및 trajectory 기반 구조 분석이 가능한 데이터가 안정적으로 생성된다 |

### Strategy OKR

| 항목 | 내용 |
| --- | --- |
| **Strategy Objective** | 사용자 행동 루프는 개입 위치에 따라 transition dynamics(확률 및 Δt 구조)가 변화하며, 이 변화는 재탐색률(RR) 및 전환 완료율(PSR)에 구조적으로 영향을 준다 |
| **K1** | 모든 사용자 행동은 time-ordered event stream으로 수집되며, event_type, timestamp, user_id, session_id 기준으로 손실 없이 기록된다 |
| **K2** | 이벤트는 event window 기반 rule/probabilistic mapping을 통해 C/I/O 상태를 우선 재구성하고, M/A₁/A₂/A₃는 초기에는 latent state로 정의한다 |
| **K3** | 상태 전이는 충분한 event density 확보 이후 정의되며, 초기에는 event sequence 기반 pseudo-transition으로 구성된다 |
| **K4** | Δt는 초기에는 inter-event interval로 정의되며, state 전이 안정화 이후에만 transition latency로 해석한다 |
| **K5** | 각 개입은 user/session 단위 exposure 여부로 정의되며, 동일 trajectory 내에서 개입 시점 이전/이후로 분리 가능한 구조를 가진다 |
| **K6** | 개입 그룹 vs 비개입 그룹은 user-level trajectory 또는 session-level sequence를 분석 단위로 정의하고, 동일 event density, trajectory 길이, state window 기준으로 정렬되어 비교 가능한 구조로 구성된다 |
| **R1** | 특정 상태 쌍($S_i→S_j$)에서 transition density가 threshold 이상으로 수렴할 경우, 해당 전이는 구조적으로 유의미한 edge로 형성된다 |
| **R2** | 개입 이후 특정 구간의 Δt 분포가 일관된 방향으로 이동하며, 행동 속도 구조 변화가 관측된다 |
| **R3** | $C→…→A_{share}→C$ cycle density 변화로 재탐색 루프(RR)의 형성 또는 억제 패턴이 드러난다 |
| **R4** | 사용자 trajectory가 반복적으로 유사한 state sequence로 수렴할 경우, behavioral attractor가 존재한다고 판단한다 |
| **R5** | 특정 state 체류 패턴이 안정화되고 전체 transition entropy가 감소하며, 시스템이 구조적으로 수렴한다 |
| **R6** | transition density 안정화 + Δt 분포 수렴 + trajectory 반복성 증가가 동시에 발생할 경우, 행동 시스템이 stable loop 구조로 형성된 것으로 판단한다 |

## 4. 퍼널별 상세 설계

퍼널 순서: **P1: 유입 및 인지 → P2: 온보딩 및 무료 이용 → P3: 결과 소비 → P4: 전환 및 결제 → P5: 공유 및 확산 → P6: 재방문 및 지속 탐색**

---

### P1: 유입 및 인지 | Test 1 — 감성 불일치 실재 여부 (Barrier Isolation)

**상태 전이**: $C → I → O$ (+$M$)
**대응 결정**: DL-01

#### 문제 정의

- 캐릭터 간의 관계성과 서사 몰입형 콘텐츠를 적극적으로 소비하는 여성 타겟은 $I_{adopt}$를 강하게 보유하고 있으나, 기존 단방향 정보 제공형 사주 서비스는 스토리텔링 및 감정적 교감 니즈를 전혀 충족시키지 못함
- 기존 플랫폼(청월당 등)은 메인스트림 감성으로 수렴 중 → 서브컬처 사용자 입장에서 $B_{brand}$ 자동 작동
- 유입 경로가 파악되지 않아 어느 커뮤니티·어떤 신호가 $I_{adopt}$를 만드는지 관찰 불가
- 타겟의 소비 성향상, '입체적인 캐릭터 간의 대립이나 강렬한 감정적 훅'이 시딩 콘텐츠에 전제되지 않으면 CTR 및 초기 유입 자체가 발생하지 않음

#### 결정 사항 (DL-01)

1. **MVP 단계에서 운세 카테고리를 연애운 단일로 설정** — 소구 포인트가 명확한 연애 카테고리에 집중하여 CTR 전환을 직접 자극하고, 단일 카테고리 설정을 통해 CTR 신호를 노이즈 없이 수집
2. **첫 진입 화면 5-7초 웹툰 형식 인트로(스킵 버튼 포함)** + 캐릭터 선택 카드(버튼 각 1장) + **실시간 캐릭터 선택 인원 표시 구조** — TTR < 15초 이내에 $M$ 형성을 조기에 유도하기 위한 최소 시각·감성 자산 확보
3. **시딩 콘텐츠에서 AI 제작 사실을 명시하고 캐릭터 이미지를 타겟 감성으로 설계** — 서브컬처 여성 사용자의 AI 창작물 거부감($B_{stigma}$)을 회피하는 대신, 진정성 있는 간극으로 전환하여 CTR 상승 효과 기대

#### 해결안

- [서브컬처 커뮤니티 사용자]가 [사주·운세 관련 대화에 반복 노출되는 상황]에서, 서브컬처 감성에 정합한 티저 콘텐츠("드디어 내 감성 사주")를 커뮤니티별로 배포하여 [$B_{brand}$ 없이 첫 진입 의도]를 만든다.
- 채널별 $UTM_{source}$ 파라미터로 반응을 분리 추적하고, 콘텐츠 유형별 CTR 차이를 관측한다.

#### 배제 대안

| 대안 | 배제 이유 |
| --- | --- |
| 총운 | 범용적이나 청월당·타이트사주와 정면 경쟁, 후킹 요소 부족으로 CTR 상승 기대 불가 |
| 총운 + 연애운 병행 | 입력 구조 복잡도 증가 + 데이터 분산으로 CTR/CRt 인과 해석 불가 |
| 정적 랜딩 (이미지 + 텍스트) | 서브컬처 감성 사용자의 첫 화면 기대값 미충족, $B_{brand}$ 즉시 상승 |
| AI 제작 미공개 | 커뮤니티 특성상 AI 그림 여부 빠르게 식별, 숨기다 발각 시 $B_{stigma}$ 급상승 |
| MVP 단계 BL 스토리 전면 배치 | MVP 단계에서 스토리 리소스 과다 |
| 애니메이션 인트로 | 개발 리소스 과다 + 이벤트 로그 수집 오류 발생 가능 |

#### 관찰 포인트

**Leading**
- 채널별 $UTM_{source}$ 수집률 0.95 미만 시 추적 구조 재설계
- 웹툰 인트로 구간 이탈 발생 여부 — `scroll_depth` 기준 이탈 집중 구간 특정
- 스킵 버튼 발생률 50% 초과 시 인트로 길이·연출 재검토
- 시딩 콘텐츠 댓글·RT 반응에서 AI 제작 언급 톤 모니터링 — 거부 반응 vs 호기심 반응 비율

**Structural**
- 랜딩 페이지 진입 이벤트(`landing_enter`) 로그 수집
- $UTM_{source}$별 유입 비율 수집 — 커뮤니티별 감성 집단 분류
- 콘텐츠 유형별 CTR 비교 — BL 시각 적용 콘텐츠 vs 미적용 콘텐츠 / AI 공개 명시 콘텐츠 vs 미명시 콘텐츠
- Barrier Isolation 실험: 그룹 A→B CTR 차이($B_{brand}$ 효과) / 그룹 B→C CTR 차이($B_{supply}$ 효과)
- 캐릭터 선택 편향 변화 관측 (선택 인원 표시 전후)

**Lagging**
- CTR 유의미한 수준 미달 시 — 카테고리 또는 시딩 훅 변경
- Barrier Isolation 결과 A→B 차이 유의미하지 않을 시 $B_{brand}$ 가설 재검토
- 랜딩 진입 후 즉시 이탈 집중 발생 시 시딩 콘텐츠와 서비스 기대 불일치 검토

#### 성공 지표

- CTR 유의미한 반응 확인 (Test 1 통과 기준)
- $UTM_{source}$ 데이터 수집 가능 상태 확인

### P2: 온보딩 및 무료 이용 | Test 2 — 행동 전환 여부

**상태 전이**: $O → M → A_1$
**대응 결정**: DL-02

#### 문제 정의

- 진입한 사용자가 $B_{total}(B_{brand} + B_{supply} + B_{stigma})$ 임계 초과로 결과 수령 전 이탈
- $B_{stigma}$: 입력 항목이 많거나 개인정보 노출 불안을 자극하는 UX 구조 → 행동 억제
- $B_{supply}$: 입력 직후 결과가 자신의 서브컬처 감성과 맞지 않을 것이라는 예측 → 완료 포기
- TTR이 15초를 초과해 행동 함수 붕괴: $P(A) \sim e^{-B_{total}}$
- 메인스트림 플랫폼과 동일한 폼 입력 방식은 감성 불일치($B_{brand}$)를 해소하지 못함

#### 결정 사항 (DL-02)

1. **강시현·한도윤 각각 서로 다른 명리학 학파 기반 분석 체계를 적용**, 운세 카테고리를 캐릭터별로 명시 — 학파 차이는 사용자에게 직접 노출하지 않고 "다른 방식의 분석이 적용되었습니다" 수준으로만 안내하여 $B_{total}$ 내 인지부하 상승을 억제
2. **무료 공개 항목**: 사용자 오행 / 올해 연애확률(숫자 블러) / 연애하기 좋은 시기(월 블러) / 연애를 도울 색(블러). 결제 시 블러 전체 해제 + 풀버전 공유 카드 활성화 — 오행 공개로 무료 결과의 실체감을 확보해 $M$을 형성하고(Loop), 블러로 $B_{supply}$를 역이용해 $CTR_{pay}$를 자극하는 One-shot 구조

> **블러 처리 주의**: 실제 분석된 내용을 블러 처리하는 것이 아닌, 알고리즘에 따라 기본 사주와 블러처리된 템플릿 페이지를 제공하여 불필요한 크레딧 소모를 억제

#### 해결안

- [랜딩 페이지에 진입한 서브컬처 감성 여성 사용자]가 [$B_{total}$ 임계 초과로 이탈하는 상황]에서, 입력 단계를 3단계 이하로 압축하고 비회원 구조 + 캐릭터 기반 온보딩 플로우를 도입하여 [$TTR \leq 15$초 + 결과 수령 완료 행동]으로 연결한다.

#### 배제 대안

| 대안 | 배제 이유 |
| --- | --- |
| 학파 차이 사용자에게 직접 노출 | 온보딩 단계에서 인지부하 급상승 유발 — $B_{stigma}$ 증가 + TTR 초과 위험 |
| 동일 데이터·톤 분기만 적용 | 캐릭터 간 해석 차이가 표면적 말투 차이에 그침 — 결제 사용자가 같은 내용을 두 번 봤다는 기분 유발 |
| 전면 무료 공개 | 이탈 최소화 가능하나 WTP 검증 불가, Test 3 소멸 |
| 결과 전체 블러 | 콘텐츠 가치 체험 전 결제 유도 = $M$ 형성 전 이탈 |
| 텍스트 요약만 무료, 상세 블러 | 일반적 페이월 구조로 서브컬처 사용자 $B_{brand}$ 재발 |

#### 관찰 포인트

**Leading**
- 단계별 이탈 이벤트(`step1_exit`, `step2_exit`, `step3_exit`) 로그 수집
- 진입 → 결과 수령 소요 시간 timestamp 수집 → $TTR_1$ 산출
- $TTR_1 \leq 15$초 미달 시 온보딩 단계 구조 재설계
- $TTR_1$ 과도하게 빠를 경우 $M$ 형성 없이 통과하는 함정 — $TTR_2$ 병행 측정 필수

**Structural**
- 단계별 완료 이벤트 로그 수집
- 단계별 체류 시간 측정 — $B_{total}$ 작동 구간 특정
- 결과 수령 완료 이벤트(`result_received`) 로그 수집 → $CR_t$ 산출
- 결과 화면 체류 시간 측정(`result_view_duration`) → $TTR_2$ 산출
- `paywall_shown` 노출 시점과 `result_view_duration` 교차 분석
- 시현·도윤 선택 편향이 80% 이상 한쪽으로 집중 시 카테고리 구성 또는 캐릭터 소개 방식 재검토

**Lagging**
- $CR_t$ 50% 미만 시 가설 폐기 — 입력 이탈 발생 구간 특정 후 해당 단계 재설계
- $TTR_2$ 60초 미만 지속 시 $M$ 형성 실패 가능성 검토
- $CVR_{pay}$ 5% 미만 시 가설 폐기
- $ARPU$ 수준 파악 — ₩990 단가 대비 LTV 구조 판단 근거로 활용

#### 성공 지표

- 단계별 이벤트 수집률 ≥ 0.95
- $CR_t \geq 50\%$
- $TTR_1 \leq 15$초 / $TTR_2$(결과 화면 체류) $\geq 60$초

### P3: 결과 소비 | Test 해당 없음 — 관찰 및 진단 구간

**상태 전이**: $A_1$ 내부 — 결과 화면 체류 + $M$ 심화 형성
**대응 결정**: DL-03

#### 문제 정의

- 결과를 수령해도 플랫폼의 의미 레이어($M$)가 사용자의 서브컬처 정체성($I_{subculture}$)과 연결되지 않으면 $J$(신념 상태) 형성 실패
- $M$ 미형성 또는 감성 공명 실패 시 $A_2$(공유)·$A_3$(결제) 전이 불가
- Semantic-aligned supply = 0 상태: 결과는 제공되지만 "내 감성 공간의 결과"라는 인식이 없으면 사실상 없는 것과 동일
- 텍스트 중심 사주 해설은 서브컬처 감성 사용자의 소비 방식과 맞지 않음 — 읽는 것이 아니라 경험하는 것을 원함

#### 결정 사항 (DL-03)

**결과 화면을 도입부 웹툰형 1:1 대화 구조 + 풀이 본문 웹소설형 서술체 + 꾸밈 템플릿 이중 포맷으로 운영**

- 도입부: 캐릭터가 사용자에게 직접 말을 거는 방식으로 $M$ 형성을 조기에 유도 → Loop 체류 구조
- 풀이 본문: 가시성과 상세도를 동시에 확보하여 결제 사용자의 소비 만족도 확보 → One-shot CTA
- 블러 해제 단일 CTA는 감정 피크 구간에 삽입하여 결제·공유 행동으로 전환

#### 해결안

명리학 기반 해석에 서브컬처 감성 언어 레이어를 덧씌운 캐릭터 중심 서사형 결과 포맷으로 체류(Loop)를 만들고 감정 피크 구간에서 블러 해제 단일 CTA를 삽입해 결제·공유 행동(One-shot)으로 전환시킨다.

#### 배제 대안

| 대안 | 배제 이유 |
| --- | --- |
| 풀이 본문 전체 웹툰형 | 결제 사용자는 상세 내용을 원하는 상태이므로 웹툰 형식 확장 시 콘텐츠 길이 과도 증가 |
| 텍스트 단일 포맷 (꾸밈 없음) | 가시성 저하, 서브컬처 감성 사용자의 소비 방식과 불일치 |
| 도입부 없이 풀이 본문 바로 시작 | 캐릭터 감성 공명($M$) 형성 없이 정보 전달 시작 — 서비스 차별성 소멸 |

#### 관찰 포인트

**Leading**
- 결과 화면 체류 시간(`result_view_duration`) 측정 — $M$ 형성 간접 신호. 60초 미만 시 서사형 포맷 또는 톤 설계 재검토
- 결과 화면 스크롤 깊이 이벤트(`scroll_depth`) 로그 수집
- 도입부(웹툰형) 구간 이탈 발생 여부 — 도입부 이탈과 풀이 본문 이탈 구간 분리 관측

**Structural**
- 캐릭터별 결과 선택 비율 추적(`character_result_select`) — 시현·도윤 편향 관측
- 결과 화면 → 다음 행동 전이 이벤트 수집(`result_to_pay` / `result_to_share` / `result_to_revisit`)
- 감정 피크 구간 CTA 클릭 시점 분포 — 체류 시간과 `pay_click` 발생 시점 상관관계 확인
- 풀이 본문 스크롤 완주율 — 꾸밈 템플릿 가시성 효과 확인

**Lagging**
- 결과 화면 → 다음 행동($A_2 \cdot A_3$) 전이율 40% 미만 시 서사형 포맷 또는 CTA 구조 재검토
- 캐릭터별 결과 선택 비율 산출 불가 시 이벤트 로그 구조 재설계
- 꾸밈 템플릿 디자인이 서브컬처 감성과 불일치 시 템플릿 재설계

#### 성공 지표

- $TTR_2$(결과 화면 체류) $\geq 60$초
- 결과 화면 → 다음 행동($A_2 \cdot A_3$) 전이율 $\geq 40\%$
- 캐릭터별 결과 선택 비율 산출 가능 상태 확인

> **메모**: 꾸밈 템플릿 디자인은 서브컬처 감성 기준으로 제작 — 메인스트림 사주 플랫폼 스타일 배제. 도입부 대화 대사는 시현·도윤 캐릭터 철학(학파 분리)과 일관성 유지 필요.

### P4: 전환 및 결제 | Test 3 — 지불 의사(WTP) 존재 여부

**상태 전이**: $A_1 → A_3$
**대응 결정**: DL-02 (가격 결정)

#### 문제 정의

- Bootstrap 단계에서 $J$는 아직 형성되지 않은 상태. 결제($A_3$)는 $J$가 아닌 $M$(의미 형성) + utility + trust 조건에서 발생
- 무료 결과로 욕구가 일부 충족되면 유료 전환 동기가 약해질 수 있음
- 유료 콘텐츠의 차별점이 체험 전에 인식되지 않으면 $CTR_{pay}$ 미발생
- 결제 진입 시점이 감정 피크와 어긋나면 $CTR_{pay}$가 낮게 형성됨

#### 결정 사항

**전체 결과 및 블러 전체 해제 단가 ₩990 단일 설정**

- 가격 > ₩1,000: WTP 미검증 상태에서 진입 허들 과도, $CVR_{pay}$ 측정값 자체가 낮아 신호 왜곡 가능
- ₩990 저가 진입점은 첫 결제 심리 저항 최소화 → $CVR_{pay}$ 상승 가능

#### 해결안

블러 잠금 구조로 유료 콘텐츠 차별점을 감정 피크 구간에 직접 노출하여 [$CTR_{pay}$ 발생 → 결제 완료 행동]으로 연결한다. WoZ 방식을 병행해 실제 WTP 존재 여부(Test 3)를 검증한다.

#### 관찰 포인트

**Leading**
- 잠금 콘텐츠 노출 이벤트(`paywall_shown`) 로그 수집
- `paywall_shown` 노출 시점과 `result_view_duration` 교차 분석
- $CTR_{pay}$ 10% 미만 시 블러 항목 구성 또는 노출 타이밍 재검토

**Structural**
- 결제 퍼널 이벤트(`premium_preview_click` → `payment_page_enter` → `pay_click` → `pay_complete`) 로그 수집
- $CTR_{pay}$ 산출: `pay_click` / `paywall_shown`
- $CVR_{pay}$ 산출: `pay_complete` / `pay_click`
- 결제 시점 timestamp 수집 → 결과 수령 후 경과 시간과 $CVR_{pay}$ 상관관계 확인
- 결제 이탈 발생 구간 특정 (`payment_page_enter` 후 이탈 집중 여부)

**Lagging**
- $CVR_{pay}$ 5% 미만 시 가설 폐기 — 가격 문제인지 콘텐츠 문제인지 분리 필요
- $ARPU$ 수준 파악 — ₩990 단가 대비 LTV 구조 판단 근거로 활용

#### 성공 지표

- $CTR_{pay} \geq 10\%$
- $CVR_{pay} \geq 5\%$
- $ARPU$ 수준 파악

### P5: 공유 및 확산 | Test 4 — 커뮤니티 바이럴 여부

**상태 전이**: $A_3 → A_{share}$
**대응 결정**: DL-05

#### 문제 정의

- 결제까지 완료한 사용자는 서비스에 대한 만족도와 정체성 공명이 가장 높은 상태이나, 공유할 수 있는 카드가 없으면 자발적 확산 행동이 발생하지 않음
- 일반 공유는 사주 개인정보 노출 부담($B_{stigma}$)으로 기피할 수 있음
- 공유가 발생하지 않으면 $A_{share}→C_{community}→I_{adopt}$ 래퍼럴 루프 자체가 형성되지 않아 Bootstrap 지속을 위한 자생적 유입 구조 확보 불가

#### 결정 사항 (DL-05)

**$A_3$(결제) 완료 사용자에게 연애운 요약 + 시현·도윤 각각의 말투로 작성된 결론 한 마디 + 각 캐릭터 특성이 드러나는 공유 카드를 자동 생성하여 제공**

- 사주 개인정보를 직접 노출하지 않는 캐릭터 감성 중심 설계로 $B_{stigma}$ 재발 방지
- $A_3→A_{share}$ 전이를 통해 래퍼럴 루프 형성

#### 배제 대안

| 대안 | 배제 이유 |
| --- | --- |
| 전체 사용자(무료 포함) 대상 공유 카드 제공 | 사주 개인정보 노출 부담($B_{stigma}$) + 커뮤니티 정서 파악 미완료 상태에서 역효과 발생 위험 |
| 결과 상세 내용 포함 공유 카드 | 사주 결과가 개인정보 수준의 민감 정보임을 감안 시 공유 기피 유발 |
| 공유 카드 미제공 | $A_3$ 완료 이후 자발적 확산 구조 부재 — 래퍼럴 루프 형성 불가 |

#### 관찰 포인트

**Leading**
- 결제 완료 직후 공유 카드 노출 이벤트(`share_card_shown`) 수집
- 공유 버튼 클릭 이벤트(`share_click`) 로그 수집 → $SR_t$ 산출
- 카드 유형별 `share_click` 발생률 비교
- $SR_t$ 30% 미만 시 카드 비주얼·문구 또는 노출 타이밍 재검토

**Structural**
- 플랫폼별 공유 완료 이벤트 수집 (트위터·아카라이브 분리)
- 공유 후 신규 유입 이벤트(`share_landing`) 로그 수집 → $UTM$ 루프 연결 추적
- `share_landing` 유입 사용자의 `landing_enter` → `result_received` 전환율 — 공유 유입 사용자 품질 확인
- 시현 카드 vs 도윤 카드 클릭률 비교
- 트위터 공유 게시물의 RT·댓글 반응 모니터링

**Lagging**
- 공유 → 신규 유입 $UTM$ 추적 가능 상태 확인
- 바이럴 루프($SR_t → C_{community}$ → 진입) 관찰 가능 상태 확인

#### 성공 지표

- 공유 이벤트 수집률 ≥ 0.95
- $SR_t \geq 30\%$
- 공유 → 신규 유입 $UTM$ 추적 가능 상태 확인
- 래퍼럴 루프($A_{share}→C_{community}$→ 진입) 관찰 가능 상태 확인

### P6: 재방문 및 지속 탐색

**상태 전이**: $A_1 / A_3 → A_2$, 이후 $J_{t+1}=F(J_t,A_t,M_t)$ 루프 안정화
**대응 결정**: DL-04

#### 문제 정의

- Bootstrap 단계에서 $A$(결제·공유)가 충분히 발생하지 않은 상태 → $J$ 업데이트 루프 미작동
- 사주 결과는 1회성 소비로 끝날 가능성이 높음 — 서비스가 자체적으로 재방문을 유도하지 못하면 외부 트리거($U$) 의존 구조 고착
- 재방문 시 새로운 콘텐츠·맥락이 없으면 재이탈 고착화 → $P_{retry} < 0.2$ 임계 도달

#### 결정 사항 (DL-04)

**결제 후 결과 열람 완료 시점에 미선택 캐릭터 등장, 다른 해석에 대한 어필**

- 세션 내 인지적 불완전성을 심어 외부 트리거 없이 `result_to_revisit`을 자연 유도
- $A_2$(재방문) 유도를 통해 $J_{initial}$ 신호를 확보
- Test 4 D7 Retention 측정을 위한 최소 내부 트리거 확보
- 추가 개발 없이 콘텐츠 분기로 구현 가능

#### 해결안

MVP 단계에서는 미선택 캐릭터 재등장("나는 다르게 봤어")을 최소 내부 트리거로 활용해 세션 내 $J$ 업데이트 동기를 심는다. Test 1~4 통과 및 $J$ 형성 신호 확인 후 $J_{t+1}=F(J_t,A_t,M_t)$ 루프 안정화를 목표로 v2에 진입한다.

#### 배제 대안

| 대안 | 배제 이유 |
| --- | --- |
| 재등장 없음 | 결제 후 서비스 종료 = 재방문 동기 소멸, D7 Retention 측정값 0 수렴 가능 |
| 푸시 알림 기반 재방문 | 비회원 구조에서 알림 수신 동의 수집 어려움 + Bootstrap 단계 개발 리소스 초과 |

#### 관찰 포인트

**Leading**
- 미선택 캐릭터 재등장 화면(`other_char_shown`) 도달률 — 결제 완료 사용자 대비 재등장 화면까지 이어지는 비율
- `other_char_shown` 도달률 낮을 시 재등장 진입 구조 재검토

**Structural**
- 재방문 이벤트(`re_visit`) 로그 수집 → D1·D7 Retention 존재 여부 우선 확인
- `other_char_shown` 이후 `re_visit` 발생 시간 간격
- 재방문 시 조회한 콘텐츠 유형 추적(`content_type_viewed`)
- 재방문 시 진입 행동 분포(다른 캐릭터 결제 vs 다른 고민 카테고리)
- 외부 트리거($U$) 시점 트래픽 스파이크 이벤트 수집

**Lagging**
- D7 Retention 존재 여부 미확인 시 재등장 대사 톤·타이밍 재검토
- 재방문 없을 시 가설 폐기 — 외부 트리거($U$) 시점 트래픽 스파이크 관찰로 전환 검토
- v2 전환 조건: Test 1~4 통과 + $A_2 \approx J_{initial}$ 신호 확인

## 7. 제약 조건 및 가드레일

| 제약 | 내용 |
| --- | --- |
| **$J$ 가정 기반 설계 금지** | 구독·장기 잔존 설계 등 $J$ 형성 가정 기반 기능은 v2 이후로 연기 |
| **Bootstrap 단계 정책** | Bootstrap 단계는 $\pi_0(A\|M, U, friction)$ 적용. $\pi_1(A\|J)$는 Bootstrap 이후 |
| **인과 근거 사용 제한** | $U$ 변수 통제 없는 단일 실험 결과를 인과 근거로 사용 금지 |
| **TTR 함정 주의** | $TTR_1 \leq 15$초 달성이 목표이나, 지나치게 빠른 결과 도달은 $M$ 형성 없이 $A_1$ 통과하는 함정. $TTR_2$ 병행 측정 필수 |
| **공유 구조** | MVP에서 $A_2$ = 재방문. 공유($A_{share}$)는 결제 완료 사용자에게만 제공. 결과 상세 내용이 아닌 캐릭터 감성 중심 구성으로 사주 개인정보 노출 최소화 |
| **블러 처리** | 실제 분석 내용을 블러 처리하는 것이 아닌, 알고리즘에 따라 기본 사주와 블러처리된 템플릿 페이지를 제공하여 불필요한 크레딧 소모 억제 |
| **WoZ 한계** | WoZ 방식은 실제 WTP 존재 여부 즉시 검증 가능하나, 운영 리소스 지속 필요 — 스케일 불가. Test 3 통과 후 자동화 전환 필요 |
| **$Fake J$ 리스크** | $A$ 발생 → $J$ 형성으로 착각하는 오류 주의. 재방문 없는 $A_2$ 발생 여부 및 커뮤니티 반응 언어로 간접 검증 |
