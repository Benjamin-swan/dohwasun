#### **MVP 단계에서 구현하기로 결정한 것**

- 랜딩페이지 구현 방식
- 재회운 단일 카테고리
- 같은 데이터·톤 분기 해석
- 무료 오행 공개 + 블러 구조
- ₩990 단일 가격
- 미선택 캐릭터 재등장

#### 퍼널 순서

**P1: 유입 및 인지 → P2 : 온보딩 및 무료 이용 → P3 : 결과 소비 → P4 : 전환 및 결제 → P5 : 공유 및 확산 → P6 : 재방문 및 지속탐색**

### **DL-01)** 유입 및 인지 (랜딩페이지 구현 방식 / 연애운 단일 카테고리)

$C→I→O$ (+$M$)

| 결정 사항 |   1. MVP 단계에서 운세 카테고리를 연애운 단일로 설정 → 소구 포인트가 명확한 연애 카테고리에 집중하여 CTR 전환을 직접 자극하고, 단일 카테고리 설정을 통해 CTR 신호를 노이즈 없이 수집
  
  2. 첫 진입 화면 5-7초 웹툰 형식 인트로(스킵 버튼 포함) → 캐릭터 선택 카드(버튼 각 1장) **+** 실시간 캐릭터 선택 인원 표시 구조로 운영 
→ 극단적 대사 대비를 통해 TTR<15초 이내에 $M$ 형성을 조기에 유도하기 위한 최소 시각, 감성 자산 확보

  3. 시딩 콘텐츠에서 AI제작 사실을 숨기지 않고, 오히려 명시하고, 캐릭터 이미지를 타겟 감성이 시각적으로 읽히도록 설계하여 마케팅에 활용
→ 서브컬쳐 여성 사용자의 AI창작물 거부감($B_{stigma}$)을 회피하는 대신, 진정성 있는 간극으로 전환하여 CTR 상승 효과 기대 |

| 대안 |   1. **총운** → 범용적이나 청월당·타이트사주와 정면 경쟁, 후킹 요소 부족으로 CTR 상승 기대 불가 → 배제
  1-2. **총운 + 연애운** 병행 → 입력 구조 복잡도 증가 + 데이터 분산으로 $CTR/CR_t$ 인과 해석 불가 → 배제

  2. 웹툰 형식은 캐릭터가 사용자에게 직접 말을 거는 구조로 $M$ 형성을 가장 직접적으로 유도하며, 애니메이션 인트로 대비 개발 리소스가 적고 이벤트 추적 구조가 단순해 Bootstrap 단계 데이터 수집 신뢰도를 높임. 캐릭터 선택 화면에 실시간 선택 인원을 노출하면 "어느 쪽이 더 많이 선택받고 있나"라는 경쟁 구도가 형성되어 선택 동기와 체류 시간을 자연스럽게 높일 수 있음. 스킵 버튼은 TTR 안전장치로 빠른 진입을 원하는 사용자 수용

### **DL-02) 온보딩 및 무료 이용 (**같은 데이터·톤 분기 해석 / 무료 오행 공개 + 블러 구조)

$O→M → A_1$

| 결정 사항 |   1. 강시현·한도윤 각각 서로 다른 명리학 학파 기반 분석 체계를 적용하고, 운세 카테고리를 캐릭터별로 명시 — 단, 학파 차이는 사용자에게 직접 노출하지 않고 "다른 방식의 분석이 적용되었습니다" 수준으로만 안내하여 $B_{total}$내 $CL_t$(인지부하) 상승을 억제하고, 결제 사용자가 같은 내용을 두 번 본다는 기분을 방지하며 캐릭터 선택의 실질적 차별성을 내부적으로 확보

  2.  무료 공개: 사용자 오행 / 올해 연애확률(숫자 블러) / 연애하기 좋은 시기(월 블러) / 연애를 도울 색(블러). 결제 시 블러 전체 해제 + 풀버전 공유 카드 활성화 — 오행 공개로 무료 결과의 실체감을 확보해 M을 형성하고(Loop), 블러로 $B_{supply}$를 역이용해 $CTR_{pay}$를 자극하는 One-shot 구조를 단일 결과 화면 안에서 완성하기 위해→ 블러 처리의 경우 실제 분석된 내용을 블러 처리하는 것이 아닌, 알고리즘에 따라 기본 사주와 블러처리된 템플릿 페이지를 제공하여 불필요한 크레딧 소모를 억제 |
| --- | --- |
| 대안 |   1. **학파 차이 사용자에게 직접 노출** → "정통 사주 vs 현대 명리학 재해석" 등 학파 설명이 온보딩 단계에서 $CL_t$ 급상승 유발 — $B_{stigma}$ 증가 + TTR 초과 위험 → 배제 / 동일 데이터·톤 분기만 적용 → 캐릭터 간 해석 차이가 표면적 말투 차이에 그침 — 결제 사용자가 같은 내용을 두 번 봤다는 기분 유발 + 실질적 결과 차별성 확보 불가, 재방문·추가 결제 동기 약화 → 배제
  
  2.  **전면 무료 공개** → 이탈 최소화 가능하나 WTP 검증 불가, Test 3 소멸 → 배제 
  
  3. **결과 전체 블러** → 콘텐츠 가치 체험 전 결제 유도 = $M$ 형성 전 이탈 → 배제 
  
  4. **텍스트 요약만 무료, 상세 블러** → 일반적 페이월 구조로 서브컬처 사용자 $B_{brand}$ 재발 → 배제
 |
| 이유 |   1. 학파 분리는 시현·도윤의 결과물이 실제로 다른 분석 체계에서 나온다는 것을 보장해 콘텐츠 차별성을 구조적으로 확보함. 사용자에게 학파 설명을 노출하지 않는 것은 온보딩 단계의 $CL_t$를 낮게 유지하면서도 결과 소비 단계에서 "왜 이렇게 다르게 봤을까"라는 궁금증을 자연스럽게 유도하기 위함. 카테고리 명시는 캐릭터 선택 화면에서 각 캐릭터의 분석 영역을 구체적으로 제시해 선택 근거를 제공

  2. 오행 공개는 "나에 대한 실제 데이터"로 무료 결과의 실체감을 확보하여 $P_{convert}$ 상승에 기여함. 블러는 $B_{supply}$를 역이용 — "거기 뭔가 있다"는 호기심으로 $CTR_{pay}$를 자극하는 구조. Loop(무료 체험으로 M 형성) → One-shot(블러 해제 CTA) 구조를 단일 결과 화면 안에서 완성 |
| 관련 지표
(행동 패턴을 볼 수 있는 도구)  |   1. $P(M∥$`result_view`) : 결과 화면 도달 후 감성 공명 형성 확률 (체류시간 ≥ 60초로 proxy 측정) / $P$(`next_action` \| $M$) : $M$형성 후 결제·공유·재방문으로 전이 확률 / `character_result_select` 비율 산출 가능 여부
  
  2. $P$ (`pay_click`∥`paywall_shown`): 블러 노출 후 결제 버튼 클릭 확률 → $CTR_{pay}≥10%$ $/ P$(`pay_complete`∥`pay_click`) : 결제 클릭 후 완료 확률 → $CVR_{pay}≥5%$ / `paywall_shown` 이벤트 수집률 ≥ 0.95 |
| 관찰 포인트
(행동패턴관찰) | **Leading**

• 시현·도윤 선택 시 각각 체류시간 차이 비교 — 어느 학파 기반 해석이 $M$ 형성에 더 기여하는지 신호
• `paywall_shown` 노출 시점과 `result_view_duration` 교차 분석 — 체류 시간이 길수록 `pay_click` 전환율이 높아지는지 상관관계 확인 
• $CTR_{pay}$ 미만 시 블러 항목 구성 또는 노출 타이밍 재검토

**Structural**
• 시현·도윤 선택 편향이 80% 이상 한쪽으로 집중 시 카테고리 구성 또는 캐릭터 소개 방식 재검토
• 결과 → 다음 행동 전이율 40% 미만 시 가설 폐기
• "다른 방식의 분석" 안내 문구 노출 구간 이탈률 모니터링
• 블러 항목별 클릭 시도 횟수 — 어느 블러를 가장 많이 건드리는지, 오래 머무르는지 관측 → 유료 전환 동기 파악
• $CVR_{pay}$ 5% 미만 시 가설 폐기

### **DL-03) 결과 소비 (콘텐츠 전달-이중포맷구조 방식 확정)**

$A_1$ 내부 — 결과 화면 체류 + $M$ 심화 형성

| 결정 사항 |   1. 결과 화면을 **도입부 웹툰형 1:1 대화 구조 + 풀이 본문 웹소설형 서술체 + 꾸밈 템플릿** 이중 포맷으로 운영 — 도입부에서 캐릭터가 사용자에게 직접 말을 거는 방식으로 $M$ 형성을 조기에 유도하고, 풀이 본문은 가시성과 상세도를 동시에 확보하여 결제 사용자의 소비 만족도를 높임 |
| --- | --- |
| 대안 |   **1. 풀이 본문 전체 웹툰형** → 결제 사용자는 상세 내용을 원하는 상태이므로 웹툰 형식 확장 시 콘텐츠 길이 과도 증가 → 소비 경험 저해 
  2. **텍스트 단일 포맷(꾸밈 없음)** → 가시성 저하, 서브컬처 감성 사용자의 소비 방식과 불일치 
  3. **도입부 없이 풀이 본문 바로 시작** → 캐릭터 감성 공명($M$) 형성 없이 정보 전달 시작 — 서비스 차별성 소멸, 이탈 위험 |
| 이유 |   1. 도입부 웹툰형 1:1 대화는 캐릭터가 사용자에게 직접 말을 거는 방식으로 $M$ 연결을 즉각 형성함. 풀이 본문은 웹소설형 서술체로 작성해 결제 사용자가 원하는 상세 정보를 길이 부담 없이 전달하되, 꾸밈 템플릿 적용으로 가시성과 독자성을 확보함. 웹툰형 전면 확장은 배제 — 결제까지 완료한 사용자는 빠른 정보 소비를 원하는 상태이므로 연출보다 내용 밀도가 우선 |
| 관련 지표
(행동 패턴을 볼 수 있는 도구)  |   1. $P(M∥$`intro_view`) : 웹툰형 도입부 소비 후 $M$ 형성 확률 (도입부 → 풀이 본문 이탈 없이 진행 비율로 proxy 측정) 
  2. $P$(`next_action`∥`full_result_view`) : 풀이 본문 완독 후 다음 행동(공유·재방문) 전이 확률 |
| 관찰 포인트
(행동패턴관찰) | **Leading**
• 도입부(웹툰형) 구간 이탈 발생 여부 — `scroll_depth` 기준 도입부 이탈과 풀이 본문 이탈 구간 분리 관측
• 도입부 이탈률 높을 시 대화 구조 또는 대사 톤 재검토

**Structural**
• 풀이 본문 스크롤 완주율 — 꾸밈 템플릿 가시성 효과 확인
• 도입부 완주 → 풀이 본문 진입 전환율
• 결과 화면 전체 체류시간(`result_view_duration`) ≥ 60초 달성 여부

### **DL-05) 공유 및 확산 (**공유 카드)

$A_3 → A_{share}$

| 결정 사항 |   1. $A_3$(결제) 완료 사용자에게 연애운 요약 + 시현·도윤 각각의 말투로 작성된 결론 한 마디 + 각 캐릭터 특성이 드러나는 공유 카드를 자동 생성하여 제공 — 사주 개인정보를 직접 노출하지 않는 캐릭터 감성 중심 설계로 $B_{stigma}$ 재발을 방지하고,  $A_3→A_{share}$ 전이를 통해 래퍼럴 루프를 형성 |
| --- | --- |
| 대안 |   1. 전체 사용자(무료 포함) 대상 공유 카드 제공 → 사주 개인정보 노출 부담($B_{stigma}$) + 커뮤니티 정서 파악 미완료 상태에서 역효과 발생 위험 → 배제  
  2. 결과 상세 내용 포함 공유 카드 → 사주 결과가 개인정보 수준의 민감 정보임을 감안 시 공유 기피 유발 + $B_{stigma}$ 재발 → 배제 
  3. 공유 카드 미제공 → $A_3$  완료 이후 자발적 확산 구조 부재 — 래퍼럴 루프 형성 불가 → 배제 |
| 이유 |   1. 결제 완료 사용자는 서비스에 대한 만족도와 정체성 공명이 가장 높은 상태로, 공유 동기가 가장 강한 시점. 캐릭터 말투가 적용된 감성 중심 카드는 사주 결과를 직접 노출하지 않으면서도 커뮤니티 내 반응을 자연스럽게 유도할 수 있음. 시현(직관적·단정적)과 도윤(논리적·가능성 중심)의 말투 차이가 카드에서도 드러나야 두 카드를 비교하는 커뮤니티 반응 발생 가능 |
| 관련 지표
(행동 패턴을 볼 수 있는 도구)  |   1. $P$(`share_click`∥`share_card_shown`) : 공유 카드 노출 후 클릭 확률
  2. $P$(`landing_enter`∥`share_landing`) : 공유 카드 클릭 후 신규 사용자 랜딩 진입 확률 |
| 관찰 포인트
(행동패턴관찰) | **Leading**
• 결제 완료 직후 공유 카드 노출 이벤트(`share_card_shown`) 수집
