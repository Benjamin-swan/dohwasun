- 최종 v1
    
    **`[1] 유입 및 인지** | Test 1 — 감성 불일치 실재 여부(Barrier Isolation)` ****
    $C_{community}$ 신호가 진입 의도로 전환되는 단계
    
    | 문제 정의  | • **캐릭터 간의 관계성과 서사 몰입형 콘텐츠를 적극적으로 소비하는 여성 타겟**은 현실의 불안을 해소할 사주 탐색 욕구($I_{adopt}$)를 강하게 보유하고 있으나, 기존의 단방향 정보 제공형 사주 서비스는 이들의 **스토리텔링 및 감정적 교감 니즈**를 전혀 충족시키지 못해 시장의 공백이 존재함.
    • 기존 플랫폼(청월당 등)은 메인스트림 감성으로 수렴 중 → 서브컬처 사용자 입장에서 $B_{brand}$ 자동 작동 → ‘콘텐츠적 경험과 과몰입’에 반응하는 핵심 타겟에게는 기존의 채널 접근 방식으로는 도달이 불가함.
    • 유입 경로가 파악되지 않아 어느 커뮤니티·어떤 신호가 $I_{adopt}$를 만드는지 관찰 불가 
    • 타겟의 소비 성향상, 이들의 시선을 즉각적으로 잡을 수 있는 ‘입체적인 캐릭터 간의 대립이나 강렬한 감정적 훅’이 시딩 콘텐츠에 전제되지 않으면 CTR 및 초기 유입 자체가 발생하지 않음. |
    | --- | --- |
    | 해결안 아이디어 |   1. [서브컬처 커뮤니티 사용자]가 [사주·운세 관련 대화에 반복 노출되는 상황]에서, 서브컬처 감성에 정합한 티저 콘텐츠("드디어 내 감성 사주")를 커뮤니티별로 배포하여 [$B_{brand}$ 없이 첫 진입 의도]를 만든다.
    
      2. [서브컬처 커뮤니티 내 사주 탐색 욕구($I_{adopt}$)가 있는 여성 사용자]가 서브컬처 감성 사용자의 감성에 일치한 “내 감성 사주”라고 인지하도록 캐릭터 기반 콘텐츠를 활용해  [$B_{brand}$없이 랜딩 페이지 진입 행동]으로 연결한다. 채널별 $UTM_{source}$ 파라미터로 반응을 분리 추적하고, 콘텐츠 유형 별 CTR 차이를 관측한다.
    
      3. 유입 경로 로그 확인 (유입 검색어 및 유입 채널 데이터)
    → $UTM_{source}$ 산출 가능 여부 확인 |
    | 장점 | • 감성 공간 신호 우선 — 기능 소개보다 "여기는 내 공간이다"는 인식이 먼저 작동 - $B_{brand}$ 해소 우선 구조
    • AI 제작 공개 + 팀의 장르 외부자 시선이 서브컬처 특유의 역반응(호기심·신기함)을 유도할 수 있음
    • 커뮤니티 1~2개 집중 시딩 (트위터 또는 아카라이브 중 하나 우선) → $I_{subculture}$ 높은 집단 정밀 유입 가능
    • $UTM_{source}$ 분리로 어느 채널·콘텐츠가 CTR을 만드는지 인과 추적 가능 |
    | 단점 | • 티저 콘텐츠 감성이 맞지 않으면 $B_{brand}$ 해소 실패 — 콘텐츠 품질에 크게 의존
    • 커뮤니티 내 광고성 콘텐츠로 인식될 경우 역효과 가능
    • 초기 시딩 커뮤니티 선택 실패 시 $I_{subculture}$ 낮은 집단 유입 위험 |
    | 관찰 포인트 &시스템 연결 | **Leading**
    • 시딩 콘텐츠 댓글·RT 반응에서 AI 제작 언급 톤 모니터링 — 거부 반응 vs 호기심 반응 비율
    • 채널별 $UTM_{source}$ 수집률 0.95 미만 시 추적 구조 재설계
    
    **Structural**
    • 랜딩 페이지 진입 이벤트(`landing_enter`) 로그 수집
    • $UTM_{source}$별 유입 비율 수집 — 커뮤니티별 감성 집단 분류
    • 콘텐츠 유형별 CTR 비교 — BL 시각 적용 콘텐츠 vs 미적용 콘텐츠 / AI 공개 명시 콘텐츠 vs 미명시 콘텐츠
    • Barrier Isolation 실험: 그룹 A→B CTR 차이($B_{brand}$ 효과) / 그룹 B→C CTR 차이($B_{supply}$ 효과)
    
    **Lagging**
    • CTR 유의미한 수준 미달 시 DL-01 v2 — 카테고리 또는 시딩 훅 변경
    • Barrier Isolation 결과 A→B 차이 유의미하지 않을 시 $B_{brand}$ 가설 재검토
    • 랜딩 진입 후 즉시 이탈 집중 발생 시 시딩 콘텐츠와 서비스 기대 불일치 검토 |
    | 성공 지표 | • CTR 유의미한 반응 확인 (Test 1 통과 기준)
    • $UTM_{source}$ 데이터 수집 가능 상태 확인 |
    
    **`[2] 온보딩 및 무료 이용 | Test 2 — 행동 전환 여부`**
    $B_{total}$ 임계 초과 없이 입력이 완료되는 단계
    
    | 문제 정의  | • 진입한 사용자가 $B_{total}(B_{brand} + B_{supply} + B_{stigma})$ 임계 초과로 결과 수령 전 이탈 
    • $B_{stigma}$: 입력 항목이 많거나 개인정보 노출 불안을 자극하는 UX 구조 → 행동 억제
    • $B_{supply}$: 입력 직후 결과가 자신의 서브컬처 감성과 맞지 않을 것이라는 예측 → 완료 포기
    • $TTR$ 이 15초를 초과해 행동 함수 붕괴: $P(A)∼e−B_{total}$ • 메인스트림 플랫폼과 동일한 폼 입력 방식은 감성 불일치 $(B_{brand}$)를 해소하지 못함 |
    | --- | --- |
    | 해결안 아이디어 | • [랜딩 페이지에 진입한 서브컬처 감성 여성 사용자]가 [$B_{total}$ 임계 초과로 이탈하는 상황]에서,
    입력 단계를 3단계 이하로 압축하고 비회원 구조 + 캐릭터 기반 온보딩 플로우를 도입하여 [$TTR≤15$초 + 결과 수령 완료 행동]으로 연결한다. 
    • 회원가입을 제거하고 고민 카테고리를 선택지로 제공해 입력 마찰을 최소화한다.
    • 서브컬쳐 기반의 인터랙티브 픽션 가눙울 도입해 사용자의 감성 불일치 $(B_{brand}$) 임계값을 낮춘다. |
    | 장점 | • 3단계 압축 + 비회원 구조로  $B_{total}$ 즉각 감소
    • 캐릭터 기반 온보딩은 감성 공명($M$ 형성)을 조기에 유도
    — $J$는 $A$ 이후 누적 형성. 온보딩 단계 목표는 $M$ 형성도
    • 선택지형 고민 입력은 "뭘 써야 하지?" 고민 제거 → 이탈 지점 감소 |
    | 단점 | • 캐릭터 온보딩은 개발 리소스가 폼 입력보다 높음 
    • 선택지가 사용자 취향과 맞지 않으면 오히려 이탈 증가
    • $TTR$≤15초와 캐릭터 연출을 동시에 달성하기 어려울 수 있음 |
    | 관찰 포인트 &시스템 연결 | **Leading**
    • 단계별 이탈 이벤트(`step1_exit`, `step2_exit`, `step3_exit`) 로그 수집
    • 진입 → 결과 수령 소요 시간 timestamp 수집 → $TTR_1$ 산출
    • $TTR_1 ≤ 15초$ 미달 시 온보딩 단계 구조 재설계
    • $TTR_1$ 과도하게 빠를 경우 M 형성 없이 통과하는 함정 — $TTR_2$ 병행 측정 필수
    
    **Structural**
    • 단계별 완료 이벤트(`step1_complete`, `step2_complete`, `step3_complete`) 로그 수집
    • 단계별 체류 시간 측정 — $B_{total}$ 작동 구간 특정
    • 결과 수령 완료 이벤트(`result_received`) 로그 수집 → $CR_t$ 산출
    • 결과 화면 체류 시간 측정(`result_view_duration`) → $TTR_2$ 산출
    
    **Lagging**
    • $CR_t 50%$ 미만 시 가설 폐기 — 입력 이탈 발생 구간 특정 후 해당 단계 재설계
    • $TTR_2$  60초 미만 지속 시 M 형성 실패 가능성 검토 — 콘텐츠 포맷 또는 온보딩 구조 재검토 |
    | 성공 지표 | • 단계별 이벤트 수집률 ≥ 0.95
    • $CR_t$ $≥ 50%$
    • $TTR≤15$초 / $TTR₂$(결과 화면 체류) $≥ 60초$ |
    
    **`[3] 결과 소비 | Test 해당 없음 — 관찰 및 진단 구간`** 
    $M→A₁$ 이후 $A₂·A₃$ 전이 조건 형성 단계
    
    | 문제 정의  | • 결과를 수령해도 플랫폼의 의미 레이어($M$)가 사용자의 서브컬처 정체성($I_{subculture}$)과 연결되지 않으면 $J$(신념 상태) 형성 실패
    • $M$ 미형성 또는 감성 공명 실패 시 $A₂$(공유)·$A₃$(결제) 전이 불가.
    $J_{t+1}=F(J_t,A_t,M_t)$는 Bootstrap 이후 단계이며,
    현 단계 목표는 $M→A$₁ 완료 후 $A₂·A₃$  전이 조건 확보
    • Semantic-aligned supply = 0 상태: 결과는 제공되지만 "내 감성 공간의 결과"라는 인식이 없으면 사실상 없는 것과 동일
    • 텍스트 중심 사주 해설은 서브컬처 감성 사용자의 소비 방식과 맞지 않음 — 읽는 것이 아니라 경험하는 것을 원함 
    • 결과 화면에서 다음 행동으로의 연결 고리가 없으면 소비 완료 순간 이탈 |
    | --- | --- |
    | 해결안 아이디어 | [결과 화면에 도달한 서브컬처 감성 여성 사용자]가 [감성 공명 없이 또는 다음 행동 트리거 없이 이탈하는 상황]에서, 명리학 기반 해석에 서브컬처 감성 언어 레이어를 덧씌운 캐릭터 중심 서사형 결과 포맷으로 체류(Loop)를 만들고 감정 피크 구간에서 블러 해제 단일 CTA를 삽입해 결제·공유 행동(One-shot)으로 전환시킨다. Loop로  $M→J$ 연결을 형성하고, One-shot으로 $A$를 발생시키는 이중 구조를 결과 화면 안에 설계한다. 강시현·한도윤의 동일 데이터·톤 분기 이중 해석 구조는 결과 화면 내 자발적 탐색을 유도해 Loop 체류 시간을 연장한다 |
    | 장점 | • 감성 언어 레이어 → "내 공간의 결과물"이라는 인식 형성 → $J$ 형성 가능성 상승
    • 결과물이 공유 가능한 형태이면 $SR_t$ 자연 발생 구조 확보
    • 명리학 기반 유지 → 정통성과 감성 동시 확보
    • 서사형 결과는 단순 텍스트 대비 체류시간 증가 가능 — Loop 구조 강화 
    • 이중 해석 구조(시현·도윤)는 결과 화면 내 자발적 탐색 유도 → 감정 피크 도달 확률 상승 
    • 블러 구조가 One-shot CTA와 결합 — 감정 피크 구간에서 결제 전환 자연 유도 |
    | 단점 | • 감성 언어 레이어 품질이 낮으면 $M \to J$ 연결 실패 — AI 생성 결과 품질 관리 필요
    • $J$ 형성 여부를 직접 측정하기 어려움 — 간접 지표(체류시간·스크롤)로만 추정 가능
    • 이중 해석 구조가 사용자에게 혼란을 줄 경우 오히려 이탈 증가 |
    | 관찰 포인트 &시스템 연결 | **Leading**
    • 결과 화면 체류 시간(`result_view_duration`) 측정 — $M$ 형성 간접 신호. 60초 미만 시 서사형 포맷 또는 톤 설계 재검토
    • 결과 화면 스크롤 깊이 이벤트(`scroll_depth`) 로그 수집 — $M$ 연결 간접 신호
    
    **Structural**
    •  캐릭터별 결과 선택 비율 추적(`character_result_select`) — 시현·도윤 편향 관측
    • 결과 화면 → 다음 행동 전이 이벤트 수집(`result_to_pay` / `result_to_share` / `result_to_revisit`)
    •  감정 피크 구간 CTA 클릭 시점 분포 — 체류 시간과 `pay_click` 발생 시점 상관관계 확인
    •  결과 화면 재방문 여부 — $M$ 형성 간접 지표
    
    **Lagging**
    •  결과 화면 → 다음 행동($A_2 ·A_3$ ) 전이율 40% 미만 시 서사형 포맷 또는 CTA 구조 재검토
    •  캐릭터별 결과 선택 비율 산출 불가 시 이벤트 로그 구조 재설계 |
    | 성공 지표 | • $TTR₂$(결과 화면 체류) $≥ 60초$
    • 결과 화면 → 다음 행동($A₂·A₃$) 전이율 $≥ 40%$
    • 캐릭터별 결과 선택 비율 산출 가능 상태 확인 |
    
    `4. 전환 및 결제 | Test 3 — 지불 의사(WTP) 존재 여부`
    ****$P(A \mid J)$가 실제로 작동하는 단계
    
    | 문제 정의  | • Bootstrap 단계에서 $J$는 아직 형성되지 않은 상태.
    결제($A₃$)는 $J$가 아닌 $M$(의미 형성) + utility + trust 조건에서 발생.
    $I_{adopt}$가 높아도 M이 $A₃$로 연결되지 않으면 결제 전환 불가
    • 무료 결과로 욕구가 일부 충족되면 유료 전환 동기가 약해질 수 있음
    • 유료 콘텐츠의 차별점이 체험 전에 인식되지 않으면 $CTR_{pay}$ 미발생
    • 결제 진입 시점이 감정 피크와 어긋나면 $CTR_{pay}$가 낮게 형성됨 |
    | --- | --- |
    | 해결안 아이디어 | [진입 후 Loop 구조에서 J가 형성된 사용자]가 [무료 결과로 만족하고 이탈하는 상황]에서, 블러 잠금 구조로 유료 콘텐츠 차별점을 감정 피크 구간에 직접 노출하여 [$CTR_{pay}$ 발생 → 결제 완료 행동]으로 연결한다. ₩990 단일 진입점으로 첫 결제 허들을 낮추고, WoZ 방식을 병행해 실제 $WTP$ 존재 여부(Test 3)를 검증한다 |
    | 장점 | • 블러 잠금 구조는 무료 체험으로 만족도를 높인 뒤 자연스럽게 결제로 유도 — 강요 없이 차별점 직접 체감
    • ₩1,990 저가 진입점은 첫 결제 심리 저항 최소화 → $CVR_{pay}$ 상승 가능

    **`5. 공유 및 확산 | Test 4 — 커뮤니티 바이럴 여부`**
    결과물이 다시 $C_{community}$를 만드는 단계 — 바이럴 루프 시작점
